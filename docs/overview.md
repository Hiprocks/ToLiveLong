# 🥗 AI 식단 관리 비서 (Diet-AI) 개발 기획서

## 1. 프로젝트 개요
- **사용자 정보**: 168cm / 60kg / 1984년생 (남성/여성 선택)
- **현재 상태**: 근육이 적은 과지방 체형
- **목표**: 근육량 증대, 지방 감소 (초정밀 식단 관리)
- **핵심 가치**: 사진 분석을 통한 영양 성분 기록 자동화 및 실시간 목표 달성률 추적

## 2. 기술 스택 (Tech Stack)
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS (Mobile First)
- **Database**: Supabase (Auth, Database, Storage)
- **AI Engine**: Gemini 1.5 Pro (Vision & Text Analysis)
- **Deployment**: Vercel

## 3. 데이터 및 영양소 정의
모든 식단 데이터는 아래 9가지 항목을 포함한다.
- **날짜**: YYYY-MM-DD (자동 생성 제목: `YYYY-MM-DD 식단관리`)
- **기본 정보**: 음식명, 섭취량(단위 포함)
- **영양 정보**: 칼로리(kcal), 탄수화물(g), 단백질(g), 지방(g), 당(g), 염분(mg)

## 4. 주요 화면 및 기능 명세

### [A] 온보딩 (설정)
- 신체 정보 입력: 키, 몸무게, 활동량, 목표 선택.
- **BMR/TDEE 자동 계산**: 입력값 기반 권장 영양소(탄/단/지/당/염분) 산출 및 저장.
- 터치 시 메인 화면 진입.

### [B] 메인 대시보드 (Today)
- **영양 요약**: 칼로리 원형 차트 및 탄/단/지 바 그래프 (단백질 강조).
- **섭취 정리 표**: 목표량 / 섭취량 / 잔여량 합계 표시.
- **오늘의 식사 리스트**:
  - [아침, 점심, 저녁, 간식] 섹션별 그룹화.
  - 한 끼에 여러 항목 섭취 시 개별 행(Row)으로 분리.
- **하단 액션**: 사진 등록(AI), 식단 등록(수동) 버튼.

### [C] 데이터 입력 (AI & Manual)
- **AI 분석**: 
  - 음식 사진(전/후) 분석하여 실시간 섭취량 추정.
  - 성분표 촬영 시 OCR을 통해 영양소 수치 자동 추출.
- **수동 입력**: 실시간 검색(includes 방식) 및 즐겨찾기 기능.

### [D] 히스토리 및 마이페이지
- 날짜별 기록 조회 및 신체 변화 추적(눈바디 검토).

## 5. 데이터베이스 구조 (Supabase)
- `profiles`: 유저 목표 정보 저장.
- `food_logs`: 일일 섭취 데이터 저장.
- `favorites`: 자주 먹는 음식 데이터.
