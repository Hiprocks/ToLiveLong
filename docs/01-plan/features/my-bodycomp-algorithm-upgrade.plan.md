# PDCA 계획서 - my-bodycomp-algorithm-upgrade

- 기능명: `my-bodycomp-algorithm-upgrade`
- 작성일: `2026-02-27`
- 현재 단계: Plan

## 1) 개요 및 목적
- My 영역에서 신체 구성(체지방률, 골격근)과 목표(린매스업) 정보를 반영해 BMR/TDEE 및 매크로 산출 로직을 개선한다.
- 식단 타입(매크로 비율)을 명시적으로 설정해 목표 칼로리/매크로 산출의 일관성을 확보한다.

## 2) 목표
- 체지방률 데이터가 있을 경우 Katch-McArdle 기반 BMR 계산을 적용한다.
- 린매스업(Body Recomposition) 목표를 독립 옵션으로 제공하고 유지 칼로리 기반 매크로 조정 로직을 확립한다.
- 식단 타입(균형/고단백/저탄고지) 선택을 매크로 산출에 반영한다.

## 3) 범위
### 포함 범위 (In Scope)
- My 프로필 입력/요약 UI: 목표/식단 타입/체지방률 반영
- BMR/TDEE/목표 칼로리/매크로 계산 로직 업데이트
- 목표-식단 타입 충돌 시 안내 문구/경고 UX
- 린매스업 목표 관련 설명/가이드 문구

### 제외 범위 (Out of Scope)
- 외부 기기 연동(InBody 등)
- 사진 기반 체지방 추정 기능
- 신규 API 엔드포인트 설계(기존 `/api/sheets/user` 범위 유지)

## 4) 기능 요구사항
- FR-1: 체지방률이 존재하면 Katch-McArdle 공식으로 BMR 계산.
- FR-2: 체지방률이 없으면 기존 Mifflin-St Jeor 공식 유지.
- FR-3: 목표 옵션에 린매스업 추가(유지 칼로리 + 단백질 비중 상향).
- FR-4: 식단 타입 선택(균형/고단백/저탄고지)을 매크로 산출에 반영.
- FR-5: 목표와 식단 타입이 상충할 경우 경고 메시지 표시 및 수정 제안.

## 5) 비기능 요구사항
- 기존 데이터와 호환 유지(필드 누락 시 기본값 적용).
- 계산 로직 변경에 따른 UI 표시 값 일관성 유지.
- 타입 안정성(TypeScript strict) 유지.

## 6) 성공 기준 (Definition of Done)
- 린매스업 목표가 독립 옵션으로 표시되고 계산 결과에 반영됨.
- 체지방률 입력 여부에 따라 BMR 계산 공식이 분기됨.
- 식단 타입 선택에 따른 매크로 결과가 변경됨.
- 목표-식단 타입 충돌 경고가 UX에 반영됨.
- 관련 변경 사항이 `docs/Context.md` 및 `docs/overview.md`에 반영됨.

## 7) 리스크 및 완화
- 리스크: 기존 사용자 데이터에 체지방률/골격근 미입력 비율이 높아 로직 분기 적용률이 낮음.
  - 완화: 미입력 시 기본 공식 유지 및 안내 문구 제공.
- 리스크: 목표-식단 타입 충돌 UX가 과도한 경고로 인식될 수 있음.
  - 완화: 설명 문구 최소화 + 선택 유지 가능 옵션 제공.

## 8) 아키텍처 고려사항
- 계산 로직은 `src/lib/calculations`에 집중(단일 진실 소스).
- My UI는 기존 컴포넌트 구조(`ProfileEditModal`, `ProfileSummarySection`, `NutritionResultCard`) 유지.

## 9) 일정/단계
- Design: 계산 분기/매크로 표 정의, UI 상호작용 명세
- Do: 계산 로직 및 UI 반영
- Check: 수동 시나리오 + lint/build
