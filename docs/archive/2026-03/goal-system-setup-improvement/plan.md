# PDCA Plan - goal-system-setup-improvement

- Feature: `goal-system-setup-improvement`
- Date: `2026-03-04`
- Roles: `PM / FE / BE / QA`
- Phase: `Plan`

## 1) Goal
- 사용자가 단순한 4개 목표 중 하나를 선택하면, 내부 계산 엔진이 개인화된 칼로리/매크로 목표를 자동 산출한다.
- 목표 계산은 시스템 로직이 100% 담당하고, AI는 수치 생성이 아닌 코칭/실행 가이드 역할만 수행한다.
- 핵심 화면(`내 정보`, `대시보드`, `AI 요약`)에서 동일한 목표 체계와 용어를 사용하도록 일관성을 확보한다.

## 2) Scope
### In Scope
- 사용자 목표를 4개로 고정: `살 빼기`, `근육 키우기`, `살 빼고 근육 키우기`, `유지하기`
- 필수/권장 입력값 체계 정리 및 검증 규칙 정의
- BMR/TDEE/목표 칼로리/매크로(단백질, 지방, 탄수화물) 계산 규칙 확정
- AI 피드백의 역할, 출력 구조, 개입 조건 정의
- 의사결정 항목(칼로리 배수 기본값, 유지하기 지방 계수, 활동 보정 규칙) 반영

### Out of Scope
- 외부 API 신규 연동
- DB 스키마 대규모 개편
- AI 모델 교체 또는 프롬프트 대규모 재설계

# 식단관리 앱 기능 기획서 (목표 체계 및 AI 어드바이스 포함)

---

## 1. 기획 목적

사용자가 직관적으로 목표를 설정하고, 내부 계산 엔진을 통해 정확한 영양 목표를 산출하며, AI는 개인화된 실행 가이드를 제공하는 구조를 설계한다.

핵심 원칙:

* 계산은 내부 로직에서 100% 처리
* AI는 수치 생성이 아닌 코칭 역할 수행
* 목표 선택은 단순화, 내부 로직은 정교화

---

# 2. 목표 체계 설계

## 2.1 사용자 목표 (4개 고정)

1. 살 빼기
2. 근육 키우기
3. 살 빼고 근육 키우기
4. 유지하기

※ 추가 세분화 없음
※ 내부 로직에서 자동 세부 전략 적용

---

# 3. 입력 정보 구조

## 3.1 필수 입력 정보

| 항목     | 형식       | 사용 목적   |
| ------ | -------- | ------- |
| 성별     | 선택       | BMR 계산  |
| 나이     | 숫자       | BMR 보정  |
| 키(cm)  | 숫자       | BMR 계산  |
| 체중(kg) | 숫자       | 매크로 계산  |
| 활동 수준  | 낮음/보통/높음 | TDEE 계산 |

활동 수준 계수:

* 낮음: 1.35
* 보통: 1.55
* 높음: 1.75

TDEE = BMR × 활동계수

---

## 3.2 권장 입력 정보 (정확도 향상)

| 항목          | 목적            |
| ----------- | ------------- |
| 체지방률(%)     | LBM 기반 단백질 계산 |
| 주간 운동시간(분)  | 활동 보정         |
| 허리둘레(cm)    | 지방 변화 추적      |
| 최근 2주 체중 변화 | 자동 강도 조절      |

---

# 4. 내부 계산 로직

## 4.1 BMR 계산식 (Mifflin-St Jeor)

남성:
BMR = 10W + 6.25H - 5A + 5

여성:
BMR = 10W + 6.25H - 5A - 161

---

## 4.2 목표별 칼로리 설정

| 목표          | 목표 칼로리           |
| ----------- | ---------------- |
| 살 빼기        | TDEE × 0.80~0.85 |
| 근육 키우기      | TDEE × 1.05~1.10 |
| 살 빼고 근육 키우기 | TDEE × 0.95~1.00 |
| 유지하기        | TDEE × 1.00      |

---

## 4.3 단백질 계산

### 체지방률 미입력
* 살 빼기: 체중 × 2.0g
* 근육 키우기: 체중 × 1.8g
* 살 빼고 근육 키우기: 체중 × 2.0g
* 유지하기: 체중 × 1.6g

### 체지방률 입력 시(권장)

LBM = 체중 × (1 - 체지방률)

* 감량/리컴프: LBM × 2.2g
* 증량: LBM × 2.0g
* 유지: LBM × 1.8g

### 단백질 상한 제한

Protein ≤ 체중 × 2.2g

---

## 4.4 지방 및 탄수화물

지방

* 감량/리컴프: 체중 × 0.8g
* 증량: 체중 × 0.9~1.0g

탄수화물:
잔여 칼로리 기준 자동 계산

Carbs(g) = (목표칼로리 - (Protein×4 + Fat×9)) / 4

---

# 5. AI 어드바이스 설계

## 5.1 AI 역할 정의

AI는 다음 기능만 수행한다:

* 현재 상태 해석
* 목표 적합성 평가
* 실행 행동 가이드 제시
* 경고 및 리스크 안내

AI는 칼로리/단백질로 계산을 수행하지 않는다.

---

## 5.2 출력 구조 (구조화된 3단 구성)

### ① 현재 상태 진단 (1문장)

사용자의 체지방, 체중 변화, 운동 여부 기반 해석

### ② 이번 주 핵심 행동 (2~3개 bullet)

* 단백질 목표 유지
* 운동 수행 기준
* 추적해야 할 지표

### ③ 주의사항 (1문장)

근손실, 지방 증가, 과도한 감량 등 리스크 안내

---

## 5.3 목표별 AI 메시지 방향

| 목표          | AI 코칭 포인트     |
| ----------- | -------------- |
| 살 빼기        | 근손실 방지, 단백질 유지 |
| 근육 키우기      | 지방 증가 속도 관리    |
| 살 빼고 근육 키우기 | 체중 정체 정상 안내    |
| 유지하기        | 변화가 없을 때 점검 유도 |

---

## 5.4 AI 개입 조건 (동적 트리거)

AI는 다음 조건에서 메시지를 강화한다:

* 단백질 달성률 80% 미만
* 체중 변화가 목표 범위 초과
* 운동 수행률 저조
* 허리둘레 증가

# 9. 추가 의사결정 항목 검토 및 권고안
## 9.1 목표 칼로리 배수 기본값 중앙값 고정 여부

### 권고: 중앙값을 기본값으로 고정하고, 자동 미세조정 로직을 추가한다.

이유:

* 초기 단계에서 변동폭을 넓게 쓰면 체중 변동성이 커져 사용자 신뢰 저하 우려가 있음
* 감량 0.80~0.85 범위 중 기본값은 0.83 권장
* 증량 1.05~1.10 범위 중 기본값은 1.07 권장
* 리컴프 0.95~1.00 범위 중 기본값은 0.97 권장

운영 방식:

* 기본값 고정
* 2주 평균 체중 변화가 목표 범위를 벗어나면 ±100~150kcal 자동 조정

※ 중앙값 고정 + 데이터 기반 보정 구조가 가장 안정적임

---

## 9.2 유지하기 지방 계수 0.85g/kg 확정 여부

### 권고: 0.85g/kg 확정 가능 (고정값 사용 권장)

근거:

* 0.8g/kg은 최소 권장선
* 0.9g/kg 이상은 증량 구간에 더 적합
* 유지 구간에서는 0.85g/kg이 호르몬 안정성과 필수 지방산 균형에 유리

결론:

* 유지하기 목표에서는 0.85g/kg 단일 고정값 사용 권장
* 범위 제공 불필요

---

## 9.3 활동량 보정 규칙 확정 여부

### 권고: 보정값은 단순화하고 과도한 세분화는 제거한다.

운영 제안:

기본 활동계수 (필수 입력)

* 낮음 1.35
* 보통 1.55
* 높음 1.75

운동시간 보정 (선택 입력 시에만 적용)

* 0~60분/주: +0.00
* 60~180분/주: +0.05
* 180~360분/주: +0.10
* 360분 이상: +0.15

NEAT은 별도 입력 받지 않음.
직업 활동량은 활동 수준에 포함.

이유:

* 사용자가 NEAT을 정확히 추정하기 어려움
* 세분화할수록 입력 오류 증가
* 체중 추세 기반 자동 보정이 더 정확

※ 활동량은 단순화, 체중 변화 데이터로 후속 보정하는 구조가 적합함
