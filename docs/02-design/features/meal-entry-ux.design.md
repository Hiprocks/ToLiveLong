# PDCA 설계서 - meal-entry-ux

- 기능명: meal-entry-ux
- 작성일: 2026-02-26
- 담당 역할: PM / FE / BE / QA
- 현재 단계: Design

## 1) 설계 목표
- Plan 문서의 목표와 범위를 UI/상태/데이터 흐름 관점에서 구체화한다.
- 기존 API 계약을 변경하지 않고 UX 개선을 구현한다.

## 2) 사용자 흐름
1. 대시보드 상단 또는 하단의 단일 `+` 버튼 클릭
2. 모드 선택 레이어 표시
3. 모드 선택
4. 해당 모달 진입 후 입력
5. 저장 버튼 클릭
6. 저장 상태에 따른 피드백 노출

## 3) 화면 구성
- 대시보드 단일 `+` 버튼(FAB)
- 모드 선택 레이어
- 수기 등록 모달
- 템플릿 등록 모달
- 사진 분석 모달
- 저장 피드백 UI

## 4) 모드 선택 레이어
- 모드: `manual`, `template`, `photo`
- 모바일 우선 터치 영역 확보
- 닫기 동작
  - 바깥 영역 탭 또는 X 버튼
  - ESC 키

## 5) 상태 모델
- 전역 상태
  - `entryMode`: `null | manual | template | photo`
  - `saveState`: `idle | saving | success | error`
- 모달 공통 초기화
  - `entryMode` 변경 시 입력값 리셋
  - `saveState`는 모달 열 때 `idle`로 초기화

## 6) 입력 검증
- `food_name`
  - trim 후 빈 문자열이면 실패
  - 실패 메시지: "음식 이름을 입력해 주세요."
- `amount`
  - 숫자 변환 후 1 이상이어야 함
  - 실패 메시지: "섭취량은 1 이상이어야 합니다."

## 7) 저장 피드백
- `saving`
  - 저장 버튼 비활성화
  - 로딩 표시
- `success`
  - 전역 성공 토스트 또는 배너
  - 모달 닫기 후 대시보드에 성공 피드백 유지
- `error`
  - 에러 배너 표시
  - API 에러 메시지 매핑

## 8) 템플릿 UX 정책
- 검색 + 선택 정책 유지
- 최근 사용 템플릿 우선 정렬
- 즐겨찾기 정책은 문서화만 하고 구현은 보류

## 9) 데이터 흐름
1. 모드 선택
2. 모달 입력
3. 검증 통과 시 저장 요청
4. 저장 결과에 따라 `saveState` 업데이트
5. 성공 시 대시보드 피드백 노출

## 10) 수동 테스트 범위
- 모드 선택 2클릭 이내 진입
- 필수값 누락 시 저장 차단
- 저장 성공 시 피드백 노출
- 저장 실패 시 에러 배너 노출
