# Changelog

Last Updated: 2026-03-10

이 문서는 전체 기능 업데이트 이력을 시간순으로 관리하는 단일 창구다.
기능 추가/변경/삭제가 발생하면 배포 여부와 관계없이 먼저 이 문서에 기록한다.

## 작성 규칙
- 신규 항목은 항상 최상단 날짜 섹션에 추가한다.
- 항목은 `Added / Changed / Fixed / Docs` 중 해당 분류에 기록한다.
- 사용자 체감 변화(화면, API, 데이터, 정책)를 1~2줄로 요약한다.
- 상세 설계/검토 내용은 PDCA 문서에 기록하고, 여기서는 링크만 남긴다.

## [2026-03-10]
### Added
- 상용 앱 대비 기능 격차 축소를 위한 기능 백로그 3종을 등록:
  - 빠른 입력 고도화(바코드/음성/반복식)
  - 주간 인사이트 리포트
  - 스트릭/리마인드 기반 습관 유지 장치
- 위 3개 항목을 `docs/Todo.md`의 `P4 (Low / 추후 고도화 검토)`로 배치해 낮은 우선순위로 관리.

### Changed
- PWA 시작 화면 배경을 검정 계열에서 오프화이트(`#f8fafc`)로 조정하고 iOS 상태바 스타일을 기본값으로 맞춰 둥근 앱 카드와 시각 톤을 일치.

### Docs
- 문서 관리 기준을 정리하고 `docs/CHANGELOG.md`를 기능 업데이트 단일 창구로 신설.
- `docs/README.md`, `docs/overview.md`, `docs/Todo.md`에 changelog 반영 규칙을 연결.

## [2026-03-06]
### Added
- 식단 등록 동선에 `AI 등록`을 추가하고 텍스트 분석 기반 prefill 흐름을 도입.
- `POST /api/analyze/text` API를 추가하여 `food_name`, `amount`, 영양성분, `intake_summary`를 반환.

### Changed
- `FoodSearchModal`에 `AI 답변 요약` 표시를 추가(저장 스키마 변경 없음).
- 공통 `LoadingOverlay`를 도입해 대시보드/히스토리/내정보/모달 로딩 UX를 일관화.
- 목표 체계를 4개(`cutting / bulking / recomposition / maintenance`)로 고정하고 계산 엔진을 재구성.
- `sedentary` 활동 계수를 1.35로 확정하고 NEAT 별도 입력을 폐지.
- 출력 단위를 칼로리 10 kcal, 매크로 5 g 반올림으로 통일.

### Fixed
- 단백질 계산에서 `recomposition` + 체지방률 입력 시 LBM 기반 계산(2.5 g/kg)을 적용하는 핫픽스 반영.

## [2026-03-03]
### Changed
- 대시보드 UI를 shadcn 기반 bento 레이아웃으로 정비.
- 칼로리 도넛/영양 진행률 색상 임계 규칙(`<=0.8`, `0.8~1.0`, `>1.0`)을 통일.
- 히스토리 영양칩을 중립 스타일로 변경하고, My 페이지 AI 메모 가독성을 개선.

## [2026-02-27]
### Added
- 린매스업 목표와 매크로 타입(균형/고단백/저탄고지)을 반영.
- 체지방률 입력 시 Katch-McArdle 기반 BMR 계산을 적용.

## [2026-02-25]
### Changed
- 식단 등록 플로우를 `+` 버튼 기반 단일 진입(수기/템플릿/사진분석)으로 단순화.
- `meal_type`을 UI/API payload에서 제거하고 일일 기록을 단일 리스트로 통합.
- 템플릿 최근 사용 우선 정책과 등록/검증 UX를 정비.

### Fixed
- analyze 응답 정규화/오류 내성을 강화하고 prebuild 테스트 게이트를 적용.
